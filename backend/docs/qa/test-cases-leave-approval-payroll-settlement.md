Every case is grounded in the implemented FastAPI routers, services, repositories, Pydantic schemas and
SQLAlchemy models. All error codes referenced here were grepped and confirmed to exist in the codebase.

Base path for every endpoint: `/api/v1`.

### 1.9 Environment

- PostgreSQL, migrated to head (`alembic upgrade head`); the four modules' tables exist with their
  documented `UNIQUE` / `CHECK` constraints.
- Every request except the anonymous cases carries `Authorization: Bearer <access_token>` bound to a
  live session row.
- Every request is org-scoped from the token's `org_id` claim; a token with no `org_id` yields
  `400 TENANT_UNRESOLVED`.
- Reset fixtures between cases (each row's Preconditions are assumed freshly seeded).

## 2. Conventions

**Error-code inventory used in this document (all verified present in `app/`):**

- Framework: `AUTH_NOT_AUTHENTICATED` (401), `AUTH_FORBIDDEN` (403), `VALIDATION_ERROR` (422),
  `NOT_FOUND` (404), `CONFLICT` (409), `TENANT_UNRESOLVED` (400), `SYSTEM_ERROR` (bulk items only).
- Leave: `LEAVE_TYPE_NOT_FOUND`, `LEAVE_TYPE_ALIAS_EXISTS`, `LEAVE_TYPE_IN_USE`, `BALANCE_NOT_FOUND`,
  `INSUFFICIENT_BALANCE`, `LEAVE_REQUEST_NOT_FOUND`, `LEAVE_OVERLAP`, `LEAVE_NOT_EDITABLE`,
  `LEAVE_NOT_CANCELLABLE`, `HOLIDAY_TEMPLATE_NOT_FOUND`, `HOLIDAY_TEMPLATE_NAME_EXISTS`,
  `HOLIDAY_ITEM_NOT_FOUND`, `ENCASHMENT_LIMIT_REQUIRED`, `EMPLOYEE_ID_REQUIRED`, `EMPLOYEE_NOT_FOUND`.
- Approvals: `APPROVAL_NOT_FOUND`, `APPROVAL_ALREADY_DECIDED`, `REJECT_REMARKS_REQUIRED`,
  `APPROVAL_FORBIDDEN_SCOPE`, `SELF_APPROVAL_NOT_ALLOWED`.
- Payroll: `PAYROLL_GROUP_NOT_FOUND`, `PAYROLL_GROUP_NAME_EXISTS`, `PAYROLL_GROUP_IN_USE`,
  `CYCLE_NOT_FOUND`, `CYCLE_EXISTS`, `CYCLE_FINALIZED`, `COMPUTED_ROW_NOT_FOUND`,
  `FINALIZED_RUN_NOT_FOUND`, `PAYROLL_ALREADY_FINALIZED`, `PAYROLL_NOT_FINALIZED`,
  `ADJUSTMENT_NOT_FOUND`, `ADJUSTMENT_EXISTS`, `EMPLOYEE_NOT_FOUND`.
- Settlements: `LOAN_ADVANCE_NOT_FOUND`, `LOAN_ADVANCE_CLOSED`, `LOAN_ADVANCE_HAS_TRANSACTIONS`,
  `ARREARS_NOT_FOUND`, `INSUFFICIENT_ARREARS`, `INVALID_TRANSACTION`, `EMPLOYEE_NOT_EXITED`,
  `PAYROLL_NOT_FINALIZED`, `SETTLEMENT_ALREADY_FINALIZED`, `EMPLOYEE_NOT_FOUND`.

**Rules that hold across all four modules:**

- A cross-tenant fetch by id is **404** (the repository's `get_by_id_in_org` returns `None`), never 403.
- A missing feature permission is **403 `AUTH_FORBIDDEN`**; a missing/expired token is
  **401 `AUTH_NOT_AUTHENTICATED`**.
- A `UNIQUE`-violating concurrent insert that loses the race surfaces as **409 `CONFLICT`**
  (SQLSTATE 23505 → `_INTEGRITY_MAP`), never a 500.
- Priorities: **P0** = security / data integrity / money (any balance, ledger, payslip or lock).
  **P1** = core flow. **P2** = validation / edge. **P3** = cosmetic.
